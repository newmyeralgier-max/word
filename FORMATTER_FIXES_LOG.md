# ГОСТ-Форматер — Дневник фиксов

## Как я увидел документ (методология)

Vision-модель на Fireworks лежит (404). Нашёл способ видеть через **позиционный анализ PyMuPDF**:

```python
import fitz
doc = fitz.open('БР_GOST.pdf')
page = doc[0]
blocks = page.get_text('dict')['blocks']
for b in blocks:
    if b['type'] == 0:
        for line in b['lines']:
            for span in line['spans']:
                print(f"x={span['origin'][0]:.0f} y={span['origin'][1]:.0f} "
                      f"size={span['size']:.0f} bold={'Bold' in span['font']} "
                      f"[{span['text'][:50]}]")
```

Это даёт x/y позицию, размер, жирность КАЖДОГО фрагмента текста. Сравнивая с оригиналом — видно ВСЁ.

Также: конвертация DOCX→PDF через LibreOffice, потом `fitz.open()` для подсчёта страниц и сравнения.

## Что обнаружено (13.04.2026)

### Оригинал vs GOST: 66 → 79 страниц (+13)

### Причина раздувания:
1. **Пустые абзацы с 1.5 интервалом** — `_handle_empty` ставил `cfg.LINE_SPACING` (1.5) на каждый пустой абзац. В оригинале 376 пустых абзацев × 1.5 интервал = +страницы
2. **Page break абзацы с 1.5** — `_insert_page_break_before` создавал параграф-разрыв с 1.5 интервалом
3. **Титульник раздут** — консультанты/Луганск уехали на 2-3 страницу вместо 1
4. **Задание раздулось** с 3 страниц до 11 — параграфы в TITLE_ZONE не получали компактный интервал

### Проблема наследования стилей:
- `_set_run_font()` перезаписывал `run.font.size` на Pt(14) даже когда в оригинале size=None (наследовался от стиля Heading = 16pt)
- Результат: заголовки H1 становились 14pt вместо 16pt
