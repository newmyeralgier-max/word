# PROBLEM_CATALOG.md — каталог проблем и решений

Каждая строка — реальная проблема, с которой столкнулись при доведении `.docx`-документа под ГОСТ 7.32-2017. Формат:

> **Симптом** в Word → **Причина** в OOXML → **Скрипт** из `pipeline/` → **Как проверить**

---

## 1. Документ открывается в формате Letter, а не A4

| | |
|---|---|
| **Симптом** | Поля плывут, всё свёрстано неправильно. |
| **Причина в OOXML** | `<w:pgSz w:w="12240" w:h="15840"/>` (Letter) вместо `<w:pgSz w:w="11906" w:h="16838"/>` (A4). |
| **Скрипт** | [`pipeline/01_pagesetup.py`](pipeline/01_pagesetup.py) |
| **Проверка** | Word: Layout → Size = A4. В файле: `<w:pgSz>` имеет `w:w="11906" w:h="16838"`. |

## 2. Номер страницы стоит на титульнике

| | |
|---|---|
| **Симптом** | Внизу первой страницы видна цифра «1». |
| **Причина в OOXML** | В `sectPr` отсутствует `<w:titlePg/>`, секция одна на весь документ. |
| **Скрипт** | [`pipeline/01_pagesetup.py`](pipeline/01_pagesetup.py) (добавляет `<w:titlePg/>` и две `<w:footerReference>`: default + first). |
| **Проверка** | На первой странице номера нет, на второй есть «2». |

## 3. Главы не начинаются с новой страницы

| | |
|---|---|
| **Симптом** | После предыдущей главы идёт следующая на той же странице. |
| **Причина в OOXML** | Нет `<w:pageBreakBefore/>` в `<w:pPr>` параграфов H1. |
| **Скрипт** | [`pipeline/02_pagebreaks.py`](pipeline/02_pagebreaks.py) (добавляет pageBreakBefore на параграфы, начинающиеся с `1.`, `2.`, ..., `СОДЕРЖАНИЕ`, `ВВЕДЕНИЕ`, и т.п., но НЕ на TOC-записи). |
| **Проверка** | Каждая глава визуально начинается на новой странице в PDF/Word. |

## 4. Подстрочный текст идёт не подстрочным (P_a, R_pu)

| | |
|---|---|
| **Симптом** | В тексте видно `P_a` или `R_pu` буквами вместо `Pₐ`/`R_PU`. |
| **Причина в OOXML** | Run без `<w:vertAlign w:val="subscript"/>`, символ `_` в обычном тексте. |
| **Скрипт** | [`pipeline/03_subscripts.py`](pipeline/03_subscripts.py) (находит паттерны `[А-Яa-z]_[а-яa-z]+`, разбивает run, на хвост ставит `vertAlign=subscript`). |
| **Проверка** | В Word буквы после `_` визуально снизу базовой линии. |

## 5. Рисунки без подписей в табличной обёртке

| | |
|---|---|
| **Симптом** | Рисунок и его подпись не центрируются и плывут по тексту. |
| **Причина** | По ГОСТ рисунок должен быть в 2-ячеечной таблице (1: рисунок, 2: подпись). В исходнике рисунки — обычные `<w:drawing>` в обычном параграфе. |
| **Скрипт** | [`pipeline/04_wrap_figures.py`](pipeline/04_wrap_figures.py) |
| **Проверка** | Каждый рисунок в Word оформлен таблицей: ячейка с картинкой + ячейка с «Рисунок N.M — ...». |

## 6. Таблицы данных с жирными заголовками и большим шрифтом

| | |
|---|---|
| **Симптом** | Таблицы выглядят несоответствующе ГОСТ. |
| **Причина** | В ячейках TNR 14pt + bold, без autofit. |
| **Скрипт** | [`pipeline/05_normalize_tables.py`](pipeline/05_normalize_tables.py) (TNR 12pt, без bold в шапке, autofit включён). Не трогает таблицы-обёртки рисунков. |
| **Проверка** | Шрифт в data-таблицах 12pt, шапка не жирная. |

## 7. Дубли номеров формул вида `(4.5)(4.5)`

| | |
|---|---|
| **Симптом** | В подписи формулы виден `(4.5)(4.5)` или `(4.9)(4.15)`. |
| **Причина** | Несколько проходов выравнивания/копирования оставили дубль `(N.M)(N.M)` в одном `<w:t>`. |
| **Скрипт** | [`pipeline/06_dedup_formulas.py`](pipeline/06_dedup_formulas.py) (regex `\\(\\d+\\.\\d+\\)\\1` → одно вхождение). |
| **Проверка** | Поиск `(N.M)(` в w:t возвращает 0 совпадений. |

## 8. Формулы перенумерованы вразнобой между главами

| | |
|---|---|
| **Симптом** | В главе 4 формулы пронумерованы `(2.4)`, `(2.7)`, `(2.8)` (как в главе 2 исходника). |
| **Причина** | Ранее формулы не перенумеровывались по главам — оставались исходные номера, которые пришли с подразделом из другой главы. |
| **Скрипт** | [`pipeline/07_renumber_formulas.py`](pipeline/07_renumber_formulas.py) (определяет текущую главу по `pageBreakBefore + "N. ТЕКСТ"`, для каждой формулы с `m:oMath` берёт ПОСЛЕДНИЙ `(X.Y)` как капшен и заменяет на `(глава.++счётчик)`. Дубли в том же параграфе удаляются.) |
| **Проверка** | Каждая глава: формулы идут `(N.1), (N.2), ...` без пропусков и дублей. |

### 8a. Чувствительность алгоритма

Старый `renumber_formulas.py` (из PR #1) брал ПЕРВЫЙ `(X.Y)` в параграфе и переименовывал только его. Если в одном параграфе была и инлайн-ссылка, и капшен — оставались мусорные хвосты типа `(4.15)(4.9)(4.15)`. Поэтому `07_renumber_formulas.py` берёт ПОСЛЕДНИЙ — это надёжнее, капшен обычно после `<w:tab/>`.

## 9. В документе остался отдельный раздел 9, хотя его контент должен быть частью раздела 10

| | |
|---|---|
| **Симптом** | В TOC есть «9. МОДЕЛИРОВАНИЕ ...», в тексте он есть как отдельная глава. |
| **Причина** | Специфика проекта: при объединении контент попал в раздел 9, должен был — в начало 10. |
| **Скрипт** | [`pipeline/08_merge_sections.py`](pipeline/08_merge_sections.py) (удаляет H1 раздела 9, переносит контент в начало раздела 10, переименовывает дублирующийся подраздел `10.1` → `10.2`). |
| **Проверка** | После прогона главы идут 1, 2, 3, ..., 8, 10. Глава 9 отсутствует. |

## 10. На страницах не показываются номера в Word (хотя в PDF из LibreOffice — есть)

| | |
|---|---|
| **Симптом** | Внизу страниц нет цифр, хотя в `footer1.xml` PAGE-field есть. |
| **Причина 1 — формат поля** | `<w:instrText>PAGE</w:instrText>` без `xml:space="preserve"`, без `\* MERGEFORMAT`, без явного шрифта на runs. LibreOffice читает, MS Word в некоторых view-режимах — нет. |
| **Скрипт 1** | [`pipeline/09_footer_pagenumber.py`](pipeline/09_footer_pagenumber.py) — переписывает `footer1.xml` в каноническом виде «1-в-1 как Word через Insert → Page Number». |
| **Причина 2 — Hide White Space** | В `settings.xml` стоит `<w:doNotDisplayPageBoundaries/>` (Word'овская опция «Скрыть пустое пространство между страницами», включается двойным кликом по разделителю страниц). При этой опции Word схлопывает межстраничные отступы и **визуально прячет колонтитулы**. |
| **Скрипт 2** | [`pipeline/10_remove_hidewhitespace.py`](pipeline/10_remove_hidewhitespace.py) — удаляет тэг. |
| **Проверка** | На стр. 2 Word показывает «2», на стр. 50 — «50». В `settings.xml` нет `<w:doNotDisplayPageBoundaries/>`. |

## 11. Пустая страница между титулом и СОДЕРЖАНИЕМ

| | |
|---|---|
| **Симптом** | Стр. 3 пустая, нумерация смещается. |
| **Причина** | Пустой параграф в `body` имеет `<w:pageBreakBefore/>`, и **следующий** параграф (СОДЕРЖАНИЕ) — тоже. Получаются два разрыва подряд. |
| **Скрипт** | [`pipeline/11_turbo_postpass.py`](pipeline/11_turbo_postpass.py), функция `fix_empty_pagebreak_paragraphs`: удаляет пустой параграф, если у следующего параграфа уже есть свой `pageBreakBefore` (так разрыв не теряется). |
| **Проверка** | После титульника сразу СОДЕРЖАНИЕ на стр. 3. |

## 12. TOC показывает устаревшие пункты `10.1.X` после объединения 9→10

| | |
|---|---|
| **Симптом** | В оглавлении видны строки `10.1.1 ... 10.1.5` и `10.1 Моделирование рабочих режимов`, хотя в тексте их уже нет (переименованы в `10.2.x`, `10.2`). |
| **Причина** | Текст пунктов TOC статично записан в `<w:hyperlink>` runs — Word не пересчитывает его до `Update Field` (F9). |
| **Скрипт** | [`pipeline/11_turbo_postpass.py`](pipeline/11_turbo_postpass.py), функция `fix_toc_stale_entries`: находит hyperlink, заменяет текст в run-последовательности **до** первого `<w:tab/>` (точки-лидеры и номера страниц сохраняются). + `inject_updatefields` ставит `<w:updateFields w:val="true"/>` чтобы Word при открытии один раз пересчитал. |
| **Проверка** | В TOC только `10.2.x`. После открытия в Word + F9 номера страниц расставляются. |

## 13. В тексте остались плейсхолдеры `«ПРОПАЛА ФОРМУЛА»`

| | |
|---|---|
| **Симптом** | Внутри математической обёртки текст «ПРОПАЛА ФОРМУЛА». |
| **Причина** | Старый ручной комментарий в `<m:oMath>`, который не убрали. |
| **Скрипт** | [`pipeline/11_turbo_postpass.py`](pipeline/11_turbo_postpass.py), функция `remove_propala_placeholder`: находит `<m:oMath>` с этим текстом и удаляет всю математическую обёртку. |
| **Проверка** | Поиск «ПРОПАЛА» возвращает 0. |

## 14. На документе остались жёлтые подсветки (маркер)

| | |
|---|---|
| **Симптом** | Случайные слова и фразы выделены жёлтым в Word. |
| **Причина** | `<w:highlight w:val="yellow"/>` в `rPr` ранов. Часто — следы рабочих пометок. |
| **Скрипт** | [`pipeline/12_clear_highlights.py`](pipeline/12_clear_highlights.py) (применять **последним**, после всех содержательных правок). |
| **Проверка** | Поиск `<w:highlight` в `document.xml` возвращает 0. |

## Как добавить новую запись

Когда находите новую проблему — порядок действий:

1. Воспроизвести симптом (открыть в Word, посмотреть, отрендерить в PDF через LibreOffice).
2. Найти причину в OOXML (распаковать .docx как ZIP, искать в `word/document.xml`, `word/footer*.xml`, `word/settings.xml`).
3. Написать скрипт `pipeline/NN_<имя>.py` с `--input`/`--output`.
4. Добавить запись в список `STEPS` в `pipeline/run_all.py`.
5. Добавить строку в `pipeline/README.md` и блок в этом каталоге.

Пишите коротко: симптом, причина, скрипт, проверка. Без воды.
