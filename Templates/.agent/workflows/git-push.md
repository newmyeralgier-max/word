# Git Push Workflow

Этот воркфлоу предназначен для безопасного сохранения вашего прогресса. Он собирает изменения, пишет красивый коммит и пушит их.

// turbo-all

1. Выполни команду `git status` чтобы посмотреть, какие файлы были изменены.
2. Выполни команду `git diff`, чтобы проанализировать изменения (используй пейджер или лимит вывода, если изменений много).
3. Убедись, что нет утечек секретов или мусорных файлов (например `.tmp/` или `venv/`). Если они есть, добавь их в `.gitignore` с помощью `run_command`.
4. Сгенерируй осмысленное сообщение для коммита согласно правилам Conventional Commits (например, `feat: ...` или `fix: ...`).
5. Выполни `git add .` (или добавь только нужные файлы).
6. Выполни `git commit -m "[твое сгенерированное сообщение]"`.
7. Выполни `git push`. (Если ветки нет в remote, выполни `git push -u origin HEAD`).
