# Word Document Expert — Скилл работы с Word-документами

## Когда активировать
- Любая работа с `.docx` файлами
- Форматирование документов (ГОСТ, IEEE, корпоративный стандарт)
- Работа с формулами (LaTeX)
- Генерация технических текстов по электроснабжению
- Применение стиля эталонного файла к рабочему
- Создание таблиц, расчётных записок, отчётов

## Режимы работы с файлами

### Режим 1: Эталон + Рабочий файл
Если пользователь предоставляет два файла:
1. **Эталонный файл** — файл, в имени которого есть слово «эталон»
   - **КАТЕГОРИЧЕСКИ ЗАПРЕЩЕНО** изменять эталонный файл
   - Используй ТОЛЬКО для чтения стилей и структуры
2. **Рабочий файл** — файл для редактирования
   - Применить правила и стиль эталона к рабочему файлу

**Алгоритм:**
```
1. Запусти execution/docx_reader.py на ЭТАЛОНЕ → получи JSON со стилями
2. Запусти execution/docx_reader.py на РАБОЧЕМ файле → получи структуру
3. Запусти execution/docx_formatter.py → применение стилей эталона к рабочему
4. Проверь результат, покажи пользователю diff
```

### Режим 2: Только рабочий файл
Если есть только рабочий файл (без эталона):
- Используй стандартный шаблон ГОСТ
- Применяй настройки из `directives/word_processing.md`

### Режим 3: Без файлов
Если файлов нет — генерируй текст по теме в структурированном Markdown.

## Правила работы с формулами

Это КРИТИЧЕСКИ ВАЖНЫЕ правила, нарушение которых неприемлемо.

### Правило 1: Исходные формулы НЕ МЕНЯТЬ
Формулы из документа берутся 1 в 1. Ты только подставляешь числовые значения в эти формулы. Не переписывай, не упрощай, не меняй обозначения.

### Правило 2: Двойной формат записи
Каждая формула записывается в ДВА вида:
1. **Обычная формула** — с подставленными числами
2. **LaTeX-блок кода** — для вставки в документ

**Пример:**

r = h · tg(α)

r = 15 · tg(45°) = 15 м

```latex
r = h \cdot \tan \alpha
```

```latex
r = 15 \cdot \tan 45^\circ = 15 \,\text{м}
```

### Правило 3: Индексы и формулы из PDF
LaTeX из .docx ты не видишь напрямую. Если есть PDF-версия — бери индексы и формулы оттуда.

### Правило 4: Запись LaTeX блоком кода
Формулы LaTeX ВСЕГДА пиши блоком кода (``` ```), а не инлайн.

## Правила форматирования текста

### Числа
- Все `.` (точки) в десятичных числах заменяй на `,` (запятые)
- Пример: `32.09` → `32,09`
- Пример: `0.75` → `0,75`

### Язык
- Работаем ТОЛЬКО на русском языке
- Все обозначения — русские (ток, напряжение, мощность, и т.д.)
- Единицы измерения — русские (кВт, кВА, А, В, Ом, м)

### Оформление текста
- Обычный текст — без `---`, `***` и прочих разделителей
- Структурирование через заголовки и списки
- Таблицы в Markdown-формате

### Формат таблиц
Пример правильной таблицы:

| Показатель | Вариант 1 | Вариант 2 |
|---|---|---|
| Установленная мощность, кВА | 2400 | 2520 |
| Коэф. загрузки | 0,63 | 0,60 |
| Потери мощности, кВт | выше | средние |

## Инструменты (Execution Scripts)

Все скрипты находятся в `execution/` и запускаются через Python 3.13:

| Скрипт | Назначение | Основные команды |
|---|---|---|
| `docx_reader.py` | Чтение .docx: параграфы, таблицы, стили, метаданные | `--file`, `--output json/text`, `--styles` |
| `docx_writer.py` | Редактирование .docx: замена текста, вставка, удаление | `--file`, `--replace`, `--insert`, `--backup` |
| `docx_formatter.py` | Форматирование по эталону или ГОСТ | `--source`, `--target`, `--standard gost` |
| `latex_converter.py` | Конвертация формул в LaTeX | `--formula`, `--fix-decimals` |
| `utils/docx_utils.py` | Утилиты: поиск, бэкап, определение эталона | (библиотека, не CLI) |

### Пример вызова:
```powershell
& "C:\Users\New-life\AppData\Local\Programs\Python\Python313\python.exe" execution/docx_reader.py --file "data/мой_документ.docx" --output text
```

## Пошаговый алгоритм работы

1. **Определи переменные** — тип документа, тема, формат выхода, детализация, стандарт, назначение
2. **Определи режим** — есть ли эталон? есть ли рабочий файл? или работаем без файлов?
3. **Прочитай директиву** — `directives/word_processing.md`
4. **Составь план** — пошаговый план для конкретной задачи
5. **Выполни** — вызови нужные скрипты из `execution/`
6. **Проверь** — корректность LaTeX, таблиц, форматирования
7. **Представь результат** — покажи пользователю в структурированном виде
