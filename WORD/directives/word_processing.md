# Директива: Работа с Word-документами по электроснабжению

Эта директива описывает стандартную операционную процедуру (SOP) для полного цикла работы с документами Microsoft Word.

## Фаза 1: АНАЛИЗ — Определение задачи

### Шаг 1.1: Сбор контекста
Определи все переменные из запроса пользователя:
- **Тип документа**: Word / Excel / PowerPoint
- **Тема**: конкретная тема электроснабжения
- **Формат выхода**: текст / таблица / формула (LaTeX) / схема / график / алгоритм
- **Уровень детализации**: кратко / стандартно / максимально подробно
- **Стандарт оформления**: ГОСТ / IEEE / корпоративный
- **Назначение**: учебное пособие / отчёт / презентация / расчётная записка

Если что-то не указано — задай уточняющий вопрос.

### Шаг 1.2: Определение входных файлов
1. Проверь наличие файлов в `data/`
2. Найди эталонный файл (имя содержит «эталон») — инструмент: `execution/utils/docx_utils.py`
3. Определи рабочий файл
4. Если есть PDF-версия — используй её для извлечения формул

## Фаза 2: ЧТЕНИЕ — Извлечение данных

### Шаг 2.1: Чтение эталона (если есть)
```powershell
& "C:\Users\New-life\AppData\Local\Programs\Python\Python313\python.exe" execution/docx_reader.py --file "data/эталон.docx" --output json --styles
```
Результат: JSON с полной информацией о стилях, шрифтах, отступах, интервалах.

### Шаг 2.2: Чтение рабочего файла
```powershell
& "C:\Users\New-life\AppData\Local\Programs\Python\Python313\python.exe" execution/docx_reader.py --file "data/рабочий.docx" --output text
```
Результат: текстовое содержимое с номерами параграфов и стилями.

### Шаг 2.3: Извлечение формул (если нужно)
Если задача связана с формулами — используй PDF-версию документа или сканируй текст на формулы.

## Фаза 3: ОБРАБОТКА — Генерация и редактирование

### Шаг 3.1: Генерация контента
В зависимости от `формат_выхода`:
- **Текст** — сгенерируй структурированный текст по теме
- **Таблица** — создай Markdown-таблицу с данными
- **Формула** — запиши в двойном формате (обычная + LaTeX-блок)
- **Алгоритм** — опиши пошагово

### Шаг 3.2: Применение правил
Обязательно проверь:
- [ ] Все десятичные точки заменены на запятые (32.09 → 32,09)
- [ ] Формулы записаны в двойном формате
- [ ] Исходные формулы не изменены (только подстановка значений)
- [ ] Все обозначения на русском языке
- [ ] Текст без лишних разделителей (`---`, `***`)

### Шаг 3.3: Форматирование документа
```powershell
& "C:\Users\New-life\AppData\Local\Programs\Python\Python313\python.exe" execution/docx_formatter.py --source "data/эталон.docx" --target "data/рабочий.docx" --standard gost
```

## Фаза 4: ПРОВЕРКА — Валидация результата

1. Проверь корректность LaTeX (все блоки компилируемы)
2. Проверь таблицы (корректное отображение в Markdown)
3. Проверь формулы (исходные не изменены, значения подставлены верно)
4. Проверь форматирование (соответствие ГОСТ/эталону)

## Фаза 5: ПРЕДСТАВЛЕНИЕ — Выдача результата

Формат выдачи:
- **Markdown-структура** с заголовками
- **Формулы** в LaTeX-блоках кода
- **Таблицы** в Markdown
- **Пояснения** по вставке в Office (если нужно)

## Обработка ошибок

| Ошибка | Решение |
|---|---|
| `ModuleNotFoundError: docx` | Установи: `pip install python-docx` |
| Файл не найден | Проверь путь, предложи пользователю указать файл |
| Эталон не определён | Спроси пользователя какой файл эталонный |
| Кодировка текста | Используй UTF-8 при чтении/записи |

## Стандартные настройки ГОСТ

Если эталон не предоставлен, применяй:
- **Шрифт**: Times New Roman, 14 пт (основной текст), 16 пт (заголовки)
- **Межстрочный интервал**: 1,5
- **Поля**: верхнее 2 см, нижнее 2 см, левое 3 см, правое 1,5 см
- **Абзацный отступ**: 1,25 см
- **Выравнивание**: по ширине (основной текст), по центру (заголовки)
- **Нумерация страниц**: внизу по центру
