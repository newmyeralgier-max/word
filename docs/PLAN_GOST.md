## 0. Краткая сводка анализа

**Объём документа.** 1149 абзацев, 103 таблицы, 162 формулы OMML, 98 изображений,
~9,5 МБ. Один section (w:sectPr) — разрывов на несколько секций нет.

**Стили (основная проблема).** Практически весь текст в стиле `Normal` (1051 из 1149).
Реально стилизовано только **2 Heading 1 и 14 Heading 3**. Остальные заголовки
разделов живут в обычном теле текста — форматер не сможет их увидеть, пока они не
помечены как заголовки.

**Заголовки-кандидаты в тексте.** По шаблону «N.` `Заглавные`» / «N.M. ...» найден
**91 абзац**, оформленный как `Normal`. Именно их нужно пометить стилями
`Heading 1/2/3`.

**Нумерация разделов сломана.** После раздела 4 сквозная нумерация срывается:
в тексте идут подряд `2.2.1`, `2.2.2`, `2.2.2.1`, `2.3.x`, потом снова `3. ОБЗОР…`,
`5. МОДЕЛИРОВАНИЕ КОЛЬЦЕВОЙ…` — фактический порядок глав расходится с ТОС.
Нужна перенумерация, см. §2.

**Подписи к рисункам и таблицам — почти отсутствуют.**

| Объект  | Всего | С подписью по ГОСТ | Без подписи / некорректно |
|---------|------:|-------------------:|--------------------------:|
| Таблицы |   103 |                 15 |                        88 |
| Рисунки |    98 |                  6 |                        92 |
| Формулы |   162 |                 30 |                       132 |

**Десятичные разделители.** 296 чисел вида `32.09` — по ГОСТ должна быть запятая
(`32,09`). Для этого уже есть ключ `docx_writer.py --fix-decimals`.

**Титульник — испорчен.** Слова слиплись и строки поехали:
- `МИНИСТЕРСТВО  НАУКИИВЫСШЕГО  ОБРАЗОВАНИЯ`
- `Захарчук А.С., АлексеенкоМ.А.` (без пробела)
- `Захарчук А.С. ,АлексеенкоМ.А.АфанасьевАлександрАлександровичЭТ-121мМоделирование …` (на аннотации)
- `© ФГБОУ ВО“Луганский …` (без пробела и кавычек нужного вида)
Нужна ручная пересборка титульной зоны (до СОДЕРЖАНИЕ) по эталону.

**Прочие замечания.**
- Заголовки с лишней точкой в конце: `4.7. Воздущные линии … (СИП).`, `4.6. Упрощённые схемы замещения линий.` — по ГОСТ точка не ставится.
- Типография слов: `Воздущные`, `БлокACVoltageSource`, `БлокControlledVoltageSource`,
  `Three-PhaseProgrammableVoltageSource`, `Три-Phase` и пр. — слитное написание
  имён блоков и отсутствие пробелов/тире. Это опечатки набора.
- Подписи таблиц часто с лишней точкой: `Таблица 4.3.`, `Таблица 4.4.` — и без названия.
- Номера формул на месте лишь у 18% формул — остальным номера надо проставить.
- TOC (toc 1/2/3) статический, страницы старые — TOC придётся перестроить после форматирования.
- Один section на весь документ → поля и ориентация одинаковы; для ГОСТ достаточно.

## 2. Стратегия (общая)

Документ нельзя «просто прогнать ГОСТ-форматером» — он не увидит заголовков
(они в `Normal`) и не придумает подписи к 92 безымянным рисункам. Поэтому идём
в три слоя:

1. **Содержательная нормализация** (полу-ручная, с участием локалки):
   исправить титульник, проставить подписи рисункам/таблицам, расшифровать
   формулы, закрыть дыры в нумерации разделов, почистить типографию.
2. **Разметка стилями** (детерминированный Python-скрипт):
   пройти по документу и перевести абзацы из `Normal` в `Heading 1/2/3`,
   `Figure Caption`, `Table Caption`, `Formula` по чётким регулярным правилам.
   Это **новый маленький скрипт** поверх логики `format_docx.py`, его напишет
   локалка или я по запросу.
3. **ГОСТ-форматирование** (детерминированный Python-скрипт):
   прогоняем итог через `format_docx.py` или `docx_formatter.py --standard gost`
   с обязательным `validate_formatter.py` на выходе.

Конечный артефакт — `data/Уч пособ Матлаб ПРАВИЛЬНО2_V4__GOST.docx`, лежит
рядом с оригиналом, оригинал не трогаем.

### ФАЗА B. Содержательная нормализация (оценка 1–2 часа с локалкой)

1. **B1. Титульник.** По скриншоту/эталону пересобрать первые ~60 абзацев
   (от `МИНИСТЕРСТВО…` до строки `Луганск 2026`). Промпт — см. §4, пункт 1.
   Правки применяет `docx_writer.py --replace` построчно или готовый скрипт
   `extract_title.py` (берёт титульник из оригинала).

2. **B2. Нумерация разделов.** Привести в порядок цепочку заголовков.
   Исходные проблемы:
   - «5.» первый раз: `5. СХЕМЫ ЗАМЕЩЕНИЯ ТРАНСФОРМАТОРОВ`
   - затем сразу «2.2.1 … 2.2.2 … 2.2.2.1 … 2.2.3 … 2.3 … 2.3.2» — это
     внутренности главы о трансформаторах
   - затем снова «3. ОБЗОР КОММУТАЦИОННЫХ БЛОКОВ»
   - затем «4. РАБОТА С POWERGUI»
   - потом уже корректно «5. МОДЕЛИРОВАНИЕ КОЛЬЦЕВОЙ…»

   Варианта два:
   - **B2a (рекомендую):** переклеить номера так, чтобы главы шли
     1→2→3→4→5→6→7 (Трансформаторы, Нагрузки+коммутаторы, POWERGUI,
     Кольцевая сеть → всё перенумеровать). Финальный список глав согласуем
     с автором (прошу список из 1 строки по каждой главе).
   - **B2b:** просто выровнять подглавы под «правильную» ветку текущей
     главы («5.2.1 …»), без сдвига номеров глав.
   Промпт см. §4, пункт 2.

3. **B3. Подписи к рисункам.** 92 безымянных изображения. Промпт для локалки
   (см. §4, пункт 3) по каждому `w:drawing`:
   - взять 1–2 абзаца контекста **до** рисунка;
   - выдать имя вида `Рисунок <N>.<M> — <краткое название>` (без точки в конце);
   - номера назначаем сквозно внутри главы: `N` = номер главы, `M` — порядковый.
   Ответ локалки пишем в файл `.tmp/figure_captions.json` (map: идентификатор
   рисунка → подпись). Потом детерминированный скрипт вставляет подписи сразу
   *под* каждым `w:drawing` новым абзацем стиля `Normal` с выравниванием CENTER
   (стиль сам наведёт `format_docx.py`).

4. **B4. Подписи/названия таблиц.** 88 таблиц без корректной подписи. Для каждой
   `w:tbl` локалка читает первую строку заголовка таблицы и 1 абзац контекста
   *над* таблицей, формирует `Таблица <N>.<M> — <название>` (над таблицей, слева,
   без абзацного отступа, без точки в конце). Промпт — §4, пункт 4. Выход:
   `.tmp/table_captions.json`.

5. **B5. Нумерация формул.** 132 формулы без номера. Две задачи:
   - расставить сквозную по главам нумерацию `(N.M)` справа от формулы;
   - проверить, что существующие 30 формул с номером не конфликтуют.
   Для этого локалка выдаёт список формул в порядке встречи и говорит, каким
   нужно добавить номер. Подсказку вставляем правым tab-stop’ом — это уже
   делает `format_docx.py::_format_formula`.

6. **B6. Десятичные разделители.** Один командой:
   ```powershell
   & python WORD/execution/docx_writer.py --file "data/Уч пособ Матлаб ПРАВИЛЬНО2_V4.docx" --fix-decimals
   ```

7. **B7. Мелкая типография.** Список лёгких замен (делаем через
   `docx_writer.py --replace`):
   - `Воздущные` → `Воздушные`;
   - `БлокACVoltageSource` → `Блок AC Voltage Source`;
   - `БлокControlledVoltageSource` → `Блок Controlled Voltage Source`;
   - `Three-PhaseProgrammableVoltageSource` → `Three-Phase Programmable Voltage Source`;
   - `Three-PhaseSource` → `Three-Phase Source`;
   - `Three-PhasePISectionLine` → `Three-Phase PI Section Line`;
   - `БлокThree-PhasePISectionLine` → `Блок Three-Phase PI Section Line`;
   - `Simulationtime` → `Simulation time`;
   - `Solveroptions` → `Solver options`;
   - `MATLABv2020b` → `MATLAB v2020b`;
   - `Tozero` → `To zero`;
   - `InitialStateSetting` → `Initial State Setting`;
   - `SimscapeElectrical` → `Simscape Electrical`.
   Полный список локалка пополняет из `.tmp/ucheb_full_text.txt`
   (промпт §4, пункт 5).

8. **B8. Снимаем точку в конце заголовков.** По ГОСТ (`RULES_GOST.md` §2)
   точка в конце заголовка НЕ ставится. Сделаем отдельным мини-скриптом:
   пройти по всем абзацам, чей текст матчит шаблон «`^\d+(\.\d+)*[.\s]+…`» ИЛИ
   который в фазе C получит стиль Heading, и если `text.endswith('.')` —
   срезать точку. Уже реализовано в `format_docx.py::_handle_heading`, но
   сейчас заголовки в `Normal`, так что сработает только после фазы C.

### ФАЗА C. Разметка стилями (оценка 30 минут, один скрипт)

Пишем новый мини-скрипт `WORD/execution/mark_structure.py` (я либо локалка).
Он **не форматирует** — только присваивает правильные `style` абзацам:

- `^(ВВЕДЕНИЕ|ЗАКЛЮЧЕНИЕ|СОДЕРЖАНИЕ|…)$` → стиль `Heading 1` (помечаем как
  структурный элемент; ГОСТ-форматер позже сделает CAPS+CENTER+NEW PAGE).
- `^\d+\.\s+[А-ЯЁ]` с длиной <200 и ВСЕМИ-ЗАГЛАВНЫМИ буквами → `Heading 1`.
- `^\d+\.\d+\.?\s+\S` → `Heading 2`.
- `^\d+\.\d+\.\d+\.?\s+\S` → `Heading 3`.
- `^Рисунок\s+\d+(\.\d+)*\s*—` → абзац с пометкой (`w:pPr/w:pStyle="Normal"`,
  логически «figure caption»; форматер поймёт по регэкспу).
- `^Таблица\s+\d+(\.\d+)*\s*—` → то же для подписей таблиц.

Скрипт должен:
1. Загружать `.docx` через `python-docx`.
2. Пропускать всё, что лежит **до** первого абзаца `ВВЕДЕНИЕ` (титульник,
   выходные данные, СОДЕРЖАНИЕ).
3. Пропускать абзацы, у которых в XML есть `m:oMath` (формулы).
4. Сохранять как `data/Уч пособ Матлаб ПРАВИЛЬНО2_V4__styled.docx`.
5. В консоль печатать: сколько абзацев стало H1/H2/H3 и список первых 20.
