# `pipeline/md_to_docx/` — сборка DOCX из Markdown

Восстановленный legacy-инструмент для Markdown→DOCX. Используется для **двух сценариев**:

1. **Создать новый DOCX из MD** (черновик на коленке → офиц. документ).
2. **Дописать MD-фрагмент в конец существующего DOCX** (новый раздел в готовый файл).

Скрипт парсит Markdown через **mistune AST** + Python-docx, формирует ГОСТ-стилизованный DOCX (TNR 14pt, поля 30/15/20/20 мм, абзацный отступ 1,25 см, полуторный интервал, нумерация формул через SEQ).

## Зависимости

```bash
pip install mistune pygments python-docx lxml
```

## Сценарий A — новый DOCX из MD

```bash
python pipeline/md_to_docx/build_docx.py \
    -i WORD/data/Исходник/draft.md \
    -o WORD/data/Готово/draft.docx \
    --fast
```

Флаги:
- `-i / --input` — входной .md файл или директория с .md (склеит все по алфавиту).
- `-o / --output` — выходной .docx (если не указан, рядом с входным).
- `--fast` — НЕ запускать Word COM для обновления TOC/формул (нужен только в Windows со встроенным Word, на Linux безопаснее `--fast`).
- `--legacy` — старый стиль подписей (`Рис.` вместо `Рисунок`, точки вместо тире).

## Сценарий B — добавить MD в существующий DOCX

```bash
python pipeline/md_to_docx/build_docx.py \
    -i WORD/data/Исходник/new_chapter.md \
    -o WORD/data/Готово/БР_GOST.docx \
    --append WORD/data/Готово/БР_GOST.docx \
    --fast
```

Флаг `--append PATH` берёт существующий .docx как базу, парсит MD из `-i`, дописывает контент **в конец** документа, сохраняет результат в `-o`. Пересборка с нуля **не** происходит — всё что было в исходном докум. (титульник, оглавление, все главы) остаётся.

> ⚠ Важно: после такого append пройди файл через `pipeline/run_all.py` чтобы привести к ГОСТ (нумерация формул, разрывы страниц, и т.д.):
> ```bash
> python pipeline/run_all.py -i WORD/data/Готово/БР_GOST.docx -o WORD/data/Готово/БР_GOST.docx
> ```

## Поддерживаемый Markdown

| MD-конструкция        | Превращается в                                                   |
| --------------------- | ---------------------------------------------------------------- |
| `# H1`                | Заголовок 1 уровня (по центру, 16pt, не жирный по умолчанию)     |
| `## H2`               | Заголовок 2 уровня (по центру, 14pt, жирный)                     |
| `### H3`              | Заголовок 3 уровня (по центру, 14pt, не жирный)                  |
| `*курсив*` / `_text_` | Курсив                                                           |
| `**жирный**`          | Жирный                                                           |
| `` `code` ``          | Inline-код (Courier New 12pt)                                    |
| ` ```python ... ``` ` | Блочный код с подсветкой через Pygments                          |
| `$...$`               | Inline-формула (LaTeX → OMML)                                    |
| `$$...$$`             | Display-формула с автонумерацией `(N.M)`                         |
| `[ссылка](url)`       | Гиперссылка                                                      |
| `- пункт` / `1. ...`  | Маркированный/нумерованный список с тире                         |
| `\| col \| col \|`    | Таблица с автонумерацией `Таблица N.M`                           |
| `![alt](path.png)`    | Рисунок с автонумерацией `Рисунок N.M`                           |

## Файлы модуля

| Файл                  | Что делает                                                       |
| --------------------- | ---------------------------------------------------------------- |
| `build_docx.py`       | Главный скрипт сборки (783 строки, парсер MD AST → docx)         |
| `word_config.py`      | ГОСТ-константы (шрифты, размеры, поля, отступы)                  |
| `word_utils.py`       | Утилиты: стили, MML2OMML-конвертер, табстопы, нумерация          |
| `latex_converter.py`  | LaTeX → MathML → OMML конвертер                                  |
| `MML2OMML.XSL`        | Microsoft Math Markup → OOXML XSLT-таблица                       |

## Известные ограничения

- Только Linux/macOS (отсутствует MS Word COM-обновление TOC). На Linux всегда указывай `--fast`. После сборки в Word нажми `Ctrl+A → F9` для обновления полей.
- LaTeX-формулы парсятся ограниченно — сложные выражения с `\\frac`, `\\int`, `\\sum` работают, но макросы (`\\newcommand`) не поддерживаются.
- При `--append` новые формулы получают номера, продолжающие сквозную нумерацию SEQ — НО номера глав в `(N.M)` берутся от количества `# H1` в MD-фрагменте, а не от позиции в целевом docx. После append всегда прогоняй `pipeline/run_all.py --only 07` для перенумерации.
